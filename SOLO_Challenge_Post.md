# 【Code with SOLO】用 TRAE SOLO 零基础爆改《三角洲行动》SBTI 干员招募战术评估（含截图分享+二维码）

## 1. 摘要
最近 SBTI 在玩家圈里很火，大家都在讨论“测测你最像哪个干员/老六”。借着这股热潮，我使用 TRAE SOLO 从零开始完整设计、开发并部署了一套基于 MBTI 逻辑的《三角洲行动》干员招募战术评估系统（SBTI）。从 15 个维度的战术化映射、30 道高网感题库、27 种干员/热梗彩蛋结果匹配，到战术 UI 统一主题配色、图片资源整理与缓存处理、Cloudflare 部署踩坑修复，再到生成带二维码的分享截图，全流程均由 SOLO 辅助完成，最终落地为一个可直接传播的前端 Web 应用。

## 2. 背景
我是一名《三角洲行动》的忠实玩家，但并不精通前端。最近各平台的性格测试很多，但大多模板化严重：题库不够“三角洲”、结果不够“懂梗”、也缺少一键分享的传播链路。

我想做一个真正贴合玩家语境的版本：题目必须包含“跑刀”“摸金”“大坝保安”“医疗兵的爹”“曼德尔砖”“六套战神”等梗；结果必须能对应官方干员与隐藏彩蛋；结算页必须能一键生成可分享图片并附二维码。于是我决定把需求交给 TRAE SOLO，用“对话式产品迭代”把它快速落地并上线。

## 3. 实践过程
整个过程我基本不写复杂代码，主要扮演“产品经理”，通过对话让 SOLO 拆解并执行。

**第一步：逻辑拆解与方案确认（企划阶段）**
- **关键 Prompt**：把原有 MBTI/SBTI 测试“爆改为《三角洲行动》版本”，题库、角色、维度、结果数量保持一致，先输出计划再实现。
- **SOLO 的表现**：它先产出规划与映射方案（对应仓库中的 [plan.md](file:///workspace/plan.md)），把 15 个维度语义改成战术语境，同时补齐 27 个结果（14 名官方干员 + 13 个热梗/隐藏彩蛋），我确认逻辑后再进入实现。

**第二步：前端页面改造与交互完善**
- **关键操作**：让 SOLO 直接在 [index.html](file:///workspace/index.html) 内实现完整答题流程：进度条、结果页布局、图片展示与角色文案。
- **踩过的坑**：结算页“图片不显示”问题，根因是结果使用了中文代号去索引图片映射，导致取不到图片。SOLO 通过反查 TYPE_LIBRARY key 的方式修复了映射逻辑，恢复了稳定出图。

**第三步：图片资源整理、缓存问题处理**
- **痛点**：梗图和官方立绘需要统一命名、统一目录，同时浏览器会缓存旧图导致更新不生效。
- **关键操作**：SOLO 使用脚本把图片重命名移动到 `image/delta_ops/`，并通过给图片 URL 追加版本号的方式解决缓存（避免“换了图但页面还是旧图”的问题）。
- **额外产出**：为了自检所有结果是否都有图，我让 SOLO 生成了一个结果总览页 [generate_all_results.html](file:///workspace/generate_all_results.html) 用于快速肉眼校验。

**第四步：主题色统一为“三角洲战术绿”并修 UI 细节**
- **需求变化**：我希望整体更像三角洲 UI：进度条、字母标识、选项边框、按钮统一成战术绿，并且 hover 不要亮到看不清字。
- **SOLO 的表现**：它先修复了 hover 过亮的问题；再将主题色改为战术绿；最后把主页主按钮的视觉统一成与进度条一致的渐变绿，保证整体一致性。

**第五步：部署到 Cloudflare 的踩坑修复**
- **踩坑**：使用 `wrangler deploy` 时静态资源上传失败，日志显示上传包含 `.git` 等无关文件，导致上传异常。
- **SOLO 的解决**：它根据日志定位到“上传目录过大/包含无关文件”的问题，并补齐 wrangler 配置，只上传必要静态资源，保证能在 Cloudflare 正常部署。

**第六步：一键分享与截图生成（传播链路）**
- **关键 Prompt**：答题结算页增加“分享按钮”，点击后生成当前页面截图，在底部加二维码，二维码跳转到本测试页。
- **SOLO 的表现**：它在结果页加入“分享结果”按钮，生成截图预览弹窗，并在截图中附带二维码，适配移动端“长按保存/转发”的分享习惯。

## 4. 成果展示
最终产出为一个纯前端 Web 应用（无需后端），包含完整答题交互、结果卡展示、可视化信息、以及一键分享传播链路。

- **测试页**：30 道《三角洲行动》网感题库 + 进度条交互（[index.html](file:///workspace/index.html)）
- **结果页**：角色匹配图、维度摘要、以及“分享结果（截图+二维码）”
- **结果总览页**：一次性查看 27 个结果的出图与文案是否完整（[generate_all_results.html](file:///workspace/generate_all_results.html)）

（发帖时建议插入截图：首页、答题中、结算页、分享弹窗、二维码落地页。）

## 5. 效果与总结
- **提效显著**：从“想法+需求”到“可用产品+部署上线”，在 SOLO 的辅助下可以非常快地迭代落地。
- **SOLO 的价值**：它不只是写代码，还能读日志排错、输出方案文档、写脚本处理素材、做 UI 统一与细节调优，覆盖了从策划到上线的完整链路。
- **复用经验**：强烈推荐工作流：先让 SOLO 输出 Plan/Spec 并确认 -> 再分步改代码 -> 素材问题直接写脚本自动化 -> 任何报错直接贴日志让 SOLO 定位根因并给出配置/修复方案。
